# v0.2 — Adaptation automatique

> Briefing technique pour la v0.2. Décrit les features à implémenter, les décisions d'architecture et les points de vigilance.

---

## Ce qui est prévu

### 1. Composants drop-in

Composants React qui s'adaptent automatiquement à l'état énergétique, sans que le dev ait à gérer la logique lui-même.

#### `<EnergyAwareImage>`

```jsx
<EnergyAwareImage
  source={{ uri }}
  highQuality={hdImage}
  lowQuality={compressedImage}
/>
```

- Affiche `highQuality` en mode `normal`
- Bascule sur `lowQuality` en mode `saver`, `critical`, ou `overheating`
- Props : `source`, `highQuality`, `lowQuality`, + toutes les props standard `Image`

#### `<EnergyAwareVideo>`

```jsx
<EnergyAwareVideo
  source={video}
  pauseOnLowPower
  reducedQualityWhenHot
/>
```

- `pauseOnLowPower` : met en pause automatiquement en mode `critical`
- `reducedQualityWhenHot` : réduit la qualité en mode `overheating`

#### `<EnergyAwareAnimation>`

```jsx
<EnergyAwareAnimation fallback={<StaticView />}>
  <LottieView ... />
</EnergyAwareAnimation>
```

- Affiche les enfants en mode `normal`
- Affiche `fallback` si `useShouldReduceAnimations()` est `true`

---

### 2. `useAdaptiveInterval`

Hook qui retourne un intervalle de polling adapté à l'état énergétique.

```js
const interval = useAdaptiveInterval({
  normal: 5000,
  saver: 30000,
  critical: 60000,
});

useEffect(() => {
  const id = setInterval(fetchData, interval);
  return () => clearInterval(id);
}, [interval]);
```

- Retourne `normal` / `saver` / `critical` selon `energyMode`
- `overheating` utilise la valeur `critical`
- Se met à jour en temps réel si l'état change

---

### 3. `<EnergyDebugOverlay>`

Overlay de debug affiché uniquement en `__DEV__`.

```jsx
{__DEV__ && <EnergyDebugOverlay position="bottom-right" />}
```

Affiche en temps réel :
- Niveau de batterie + état de charge
- Mode énergie (`energyMode`)
- État thermique
- Low power mode on/off
- FPS recommandé

Props : `position` (`top-left` | `top-right` | `bottom-left` | `bottom-right`), `style`

## Architecture

### Composants

Les composants drop-in sont de simples wrappers React qui consomment les hooks existants — pas de nouveau code natif nécessaire.

Structure envisagée :

```
src/
├── components/
│   ├── EnergyAwareImage.tsx
│   ├── EnergyAwareVideo.tsx
│   ├── EnergyAwareAnimation.tsx
│   └── EnergyDebugOverlay.tsx
├── hooks/
│   ├── useAdaptiveInterval.ts     ← nouveau
│   └── ... (existants)
└── index.tsx                      ← exporter les nouveaux composants/hooks
```

### `EnergyProvider` — décision reportée à v1.0

Plusieurs composants sur le même écran créeront chacun leur propre abonnement natif. Le ref-counting natif (déjà en place) évite les doublons d'observers côté natif, mais il y aura autant d'abonnements `NativeEventEmitter` que de composants. Acceptable pour v0.2 — le provider sera introduit en v1.0.

---

## Points de vigilance

- **`EnergyAwareVideo`** : ne pas coupler directement à une lib vidéo — accepter un render prop ou des children pour rester agnostique (`expo-video`, `react-native-video`, etc.)
- **`EnergyDebugOverlay`** : utiliser `StyleSheet.absoluteFillObject` + `pointerEvents="none"` pour ne pas bloquer les interactions
- **Exports** : penser à exporter les nouveaux composants dans `src/index.tsx` et vérifier que les types sont bien générés par bob

---

## Ce qui N'est PAS dans cette version

- Pas de `EnergyProvider` / Context React — reporté à v1.0
- Pas de plugins pour libs tierces (Reanimated, FlashList, etc.) — prévu v1.0
- Pas de wrapper pour tâches lourdes (`useEnergyAwareTask`) — prévu v1.0
